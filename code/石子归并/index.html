Problem 石子归并

时间限制 1000 ms1000 \ ms1000 ms
内存限制 64 MB64 \ MB64 MB

题目描述

考虑下面这个经典的石子归并模型：有 nnn 堆石子，排成一行，每次可以选择相邻的两堆石子合并成一堆，花费的代价为两堆石子的重量之和，求将所有的石子合并成一堆需要的最小花费。 现在加入一个限制，设合并时两堆石子重量为 aaa 和 bbb ，如果 min⁡(a,b)>=10\min(a,b)>=10min(a,b)>=10，那么当且仅当 max⁡(a,b)\max(a,b)max(a,b) 不能被 min⁡(a,b)\min(a,b)min(a,b) 整除时，两堆石子才能合并；对于 min⁡(a,b)<10\min(a,b)<10min(a,b)<10 的情况则没有此限制。 求加入此限制的石子归并模型的最小花费，若无法将所有石子合并成一堆则输出 −1-1−1。

输入数据

第一行为一个正整数 nnn，n<=300n<=300n<=300 接下来一行为 nnn 个正整数，代表每一堆石子的重量，每堆石子的重量不超过 300300300

输出数据

一个正整数代表最小花费，或者 −1-1−1 代表无法合并成一堆
