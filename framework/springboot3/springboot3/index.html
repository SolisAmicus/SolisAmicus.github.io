常用注解

组件注解

  • @Configuration：用于定义配置类，其中包含应用程序的配置信息。

  • @SpringBootConfiguration：@Configuration 的特殊化，Spring Boot\text {Spring Boot}Spring Boot 中的主配置类通常使用这个注解。

  • @Bean：在 Spring\text {Spring}Spring 容器中定义 Bean\text {Bean}Bean，根据配置方法的返回值创建该 Bean\text {Bean}Bean。
  • @Scope：用于指定 Bean\text {Bean}Bean 的作用域，如单例、原型等。

  • @Controller：用于标记控制器类。

  • @Service：用于标记服务类。

  • @Repository ：用于标记数据访问层的类。

  • @Component：是通用的组件注解，用于标记任何 Spring Bean\text {Spring Bean}Spring Bean。

  • @ComponentScan：用于启用组件扫描，Spring Boot\text {Spring Boot}Spring Boot 将扫描指定的包以查找带有 @Component 注解的类并注册它们为 Bean\text {Bean}Bean。

  • @Import：用于导入其他配置类，将它们的 Bean\text {Bean}Bean 定义合并到当前配置类中。

条件注解

条件注解允许基于特定条件来决定是否应该创建或注册Bean\text {Bean}Bean。条件注解通常用于解决根据应用程序的环境、配置或其他因素来选择性地加载 Bean\text {Bean}Bean 的需求。以下是一些常见的条件注解：

  • @ConditionalOnClass：当类路径中存在指定的类时，才会创建 Bean\text {Bean}Bean。
  • @ConditionalOnMissingClass：当类路径中不存在指定的类时，才会创建 Bean\text {Bean}Bean。
  • @ConditionalOnBean：仅当容器中存在指定类型的 Bean\text {Bean}Bean 时，才会创建 Bean\text {Bean}Bean。
  • @ConditionalOnMissingBean：仅当容器中不存在指定类型的 Bean\text {Bean}Bean 时，才会创建 Bean\text {Bean}Bean。
  • @ConditionalOnProperty：仅当指定属性存在并具有特定值时，才会创建 Bean\text {Bean}Bean。
  • @ConditionalOnExpression：通过 SpEL\text {SpEL}SpEL 表达式来确定是否创建 Bean\text {Bean}Bean。
  • @ConditionalOnResource：当类路径中存在指定资源文件时，才会创建 Bean\text {Bean}Bean。
  • @ConditionalOnWebApplication：仅在 Web 应用程序环境下才会创建 Bean\text {Bean}Bean。
  • @ConfigurationProperties ：用于将外部配置文件中的属性值绑定到 Bean\text {Bean}Bean 的属性。（导入自己的包）
  • @EnableConfigurationProperties ：用于将外部配置文件中的属性值注册到 Bean\text {Bean}Bean 的属性。这个注解通常在主应用程序类上使用。（导入第三方包）

YAML配置文件

YAML \text {YAML }YAML （YAML Ain’t Markup Language\text {YAML Ain't Markup Language}YAML Ain’t Markup Language）：可读的数据序列化格式，通常用于配置文件和数据交换。YAML \text {YAML }YAML  配置文件使用简单的文本格式，通常以 .yml 或 .yaml 扩展名保存，以表示层次结构的数据。

  • 注释以#符号开头，可以用来添加注释说明。
  • 层次结构使用空格缩进表示，通常是两个空格或四个空格，用于表示嵌套关系。
  • 键值对使用冒号:分隔
  • 列表使用短横线-开头
  • 布尔值可以表示为true或false
  • 多行文本可以使用|或>符号表示

日志配置

日志配置是指确定日志记录的目标、格式、级别和其他属性的设置。典型的配置包括：

  • 日志级别：决定哪些日志消息应该被记录，通常包括调试、信息、警告、错误等级别。
  • 日志输出目标：确定日志消息应该被发送到哪里，如控制台、文件、数据库等。
  • 日志格式：指定日志消息的输出格式，包括时间戳、日志级别、消息内容等。
  • 日志滚动策略：定义日志文件的滚动方式，如按文件大小、日期等切割日志文件。

日志门面是一个抽象接口或API，它提供了一种方式来记录日志消息，但不负责实际的日志记录。它允许开发人员在应用程序中编写日志记录代码，而无需关心底层的日志实现细节。

日志实现是实际执行日志记录的组件。它根据配置将日志消息写入所选的目标，如文件、数据库、控制台等。

Spring Boot\text {Spring Boot}Spring Boot 使用了一个灵活且高度可配置的日志实现机制，它默认集成了 SLF4J\text {SLF4J}SLF4J（Simple Logging Facade for Java\text {Simple Logging Facade for Java}Simple Logging Facade for Java）作为日志门面，以及 Logback\text {Logback}Logback 作为默认的日志实现。

  • 日期
  • 时间（毫秒级）
  • 消息级别
  • 进程 ID\text {ID}ID
  • 消息名称：通常是类的全名。
  • 消息内容

日志级别

  • TRACE\text {TRACE}TRACE：用于记录应用程序的详细跟踪信息。

  • DEBUG\text {DEBUG}DEBUG：用于记录应用程序的调试信息，通常在开发和测试阶段使用。

  • INFO\text {INFO}INFO：用于记录应用程序的一般信息，通常用于生产环境的日志记录。

  • WARN\text {WARN}WARN：用于记录可能的问题或不寻常的情况，但不会导致应用程序中断。通常表示潜在的问题，但应用程序仍然可以正常运行。

  • ERROR\text {ERROR}ERROR：用于记录错误和异常，表示应用程序遇到了一个严重的问题，可能会导致应用程序崩溃或无法正常工作。

  • FATAL\text {FATAL}FATAL：非常严重的错误，通常表示应用程序将无法继续运行。

要更改日志级别，一般通过两种方式：

  1. 修改相应类文件
