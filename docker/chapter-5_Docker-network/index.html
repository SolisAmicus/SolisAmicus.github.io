检查从本地计算机到目标 IP\text {IP}IP 地址之间的网络路径：

每一行代表了路径中的一个跃点，以及跃点对应的 IP\text {IP}IP 地址和响应时间（以毫秒为单位）。以下是每个部分的解释：

  • 第一列是跃点的序号。例如，1?\text {1?}1? 表示本地主机（您的计算机），1\text {1}1 表示第一个跃点，2\text {2}2 表示第二个跃点，以此类推。
  • 第二列是跃点对应的 IP\text {IP}IP 地址。
  • 第三列是从本地计算机到该跃点的往返响应时间。

路由跃点的数量可能会因为网络的配置和路径的复杂性而有所不同。每个"no reply\text {no reply}no reply“表示在此跃点上没有响应，可能是因为该跃点未回复或者设置了防火墙规则阻止了回复。一旦网络路径中的某个路由器或设备不回应，后续的跃点将显示为”no reply\text {no reply}no reply"。

计算机发送第一个数据包到目标 IP\text {IP}IP 地址，数据包的 TTL（生存时间）设置为 111。第一个路由器（跃点）收到数据包后，将 TTL\text {TTL}TTL 减 111， TTL\text {TTL}TTL 减少到 000，路由器丢弃数据包并发送一个"Time Exceeded\text {Time Exceeded}Time Exceeded“消息回复给发送者，表示数据包已超时。计算机接收到”Time Exceeded\text {Time Exceeded}Time Exceeded“回复后，知道第一个跃点的 IP\text {IP}IP 地址和响应时间。计算机发送第二个数据包，TTL\text {TTL}TTL 设置为 222，重复上述过程，以获取第二个跃点的信息。计算机逐渐递增 TTL\text {TTL}TTL，重复发送数据包，以获取每个跃点的 IP\text {IP}IP 地址和响应时间。如果数据包到达目标 IP\text {IP}IP 地址，则目标主机会发送回复，显示网络路径的最后一跃点。否则，”traceroute\text {traceroute}traceroute“将显示”no reply\text {no reply}no reply“表示没有回复。在某些情况下，如果网络路径太长，数据包可能会因为超过最大传输单元（PMTU）而被丢弃，”traceroute\text {traceroute}traceroute“将显示”Too many hops\text {Too many hops}Too many hops"并提供相关信息。
